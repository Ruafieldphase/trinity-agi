# Copilot Chat 대화 저장 정책 (Conversation Storage Policy)

작성일: 2025-10-30
문서 목적: VS Code GitHub Copilot Chat의 대화/요약을 어디에, 어떤 형식으로, 어떤 범위로 저장할지 표준화하여 중복/소음/리스크를 줄이고 추적성을 높입니다.

## 1) 기본 원칙

- 단일 진실 소스(SSOT)는 내부 시스템입니다: 장문 대화 원문, 세부 로그, 실험 산출물, 민감 정보는 내부 저장소에만 보관합니다.
- GitHub는 얇은 색인 계층(Index)으로 사용합니다: 코드와 직접 연결되는 결정/근거/작업만 요약해 남기고, 자세한 내용은 내부 문서/대시보드 링크로 대체합니다.
- 자동 업로드 최소화: “전량 대화/요약 자동 업로드”를 지양하고, 사람이 의도적으로 ‘결정·근거·작업’만 게시합니다.

## 2) 무엇을 어디에 남길지

- 내부 시스템 전용
  - 장문 대화(원문), 상세 로그/메트릭, 실험 원자료, PII/시크릿 포함 내용, 임시 메모
- GitHub에도 요약 필수(색인 목적)
  - 설계/제품 결정과 트레이드오프, SLA/성능에 영향, 릴리즈/마이그레이션/리팩터링 등 코드 변화에 직접적 근거가 되는 사안
  - 형식: 5~10줄 요약 + 근거(2~3개 불릿) + 내부 링크 + 작업 항목(담당/기한)
- GitHub만(내부 참조 불필요 시)
  - 소규모 코드 작업 관리, 단순 버그/타이포 등

## 3) VS Code 설정 가이드 (권장)

> 실제 설정 명칭은 VS Code/확장 버전에 따라 다를 수 있습니다. 아래 경로 기준으로 유사 옵션을 찾아 비활성화/축소를 권장합니다.

- Chat 히스토리 저장 범위 축소
  - VS Code 설정 → Extensions → GitHub Copilot Chat → Chat History 관련 옵션 → 조직 정책에 맞게 Off 또는 최소화
- 자동 공유/게시 방지
  - “Create Issue/PR로 자동 게시” 류의 단축 액션은 기본 사용 금지. 필요 시 수동으로 요약을 다듬은 뒤 게시
- 텔레메트리/클립보드/스니펫 공유 범위 검토
  - 조직 정책에 따라 VS Code의 Telemetry, Copilot 관련 고급 옵션을 최소화
- 팀 합의: 기본값은 ‘내부 저장소에만 저장’으로, GitHub 게시 시에는 결정/작업/링크만

## 4) 운영 워크플로

1. Copilot Chat으로 탐색/토론/브레인스토밍
2. 내부 저장소에 장문 리포트·증거(로그, 그래프, 샘플)를 정리
3. GitHub Issue/PR에는 다음 최소 블록만 게시
   - Summary: 3~5줄
   - Decision: 한 줄 핵심 + 근거 2~3개 불릿
   - Evidence: 내부 문서/대시보드 링크
   - Action Items: 체크박스 + 담당자 + Due
4. 주 1회(또는 마일스톤 종료 시) GitHub Discussions에 ‘주간 결정/핵심 지표/이슈’ 하이라이트만 게시(세부는 내부 링크)

## 5) 보안/개인정보

- PII/시크릿/토큰/고객 데이터가 Chat 로그나 GitHub에 유출되지 않도록 필터링
- 내부 링크는 최소 권한 원칙(접근 권한 확인), 외부 공개 금지
- 장문 원본은 내부에만, GitHub에는 요약+링크만

## 6) 측정과 개선

- 노이즈 지표: 이슈/PR 댓글 볼륨 대비 조회수·참조수 비율
- 리드타임: 결정→작업 생성→PR 머지까지 걸린 시간
- 회고: 롤업 게시물의 링크 클릭률/필요한 정보 찾기 시간

## 7) FAQ

- Q: Copilot Chat 히스토리를 완전 차단해야 하나요?
  - A: 개발 편의성 측면에서 ‘로컬/계정 히스토리’는 허용하되, 대외 게시/자동 업로드는 금지하고 내부 저장 우선 정책을 지키는 것을 권장합니다.
- Q: 어떤 경우에 GitHub에 바로 남기나요?
  - A: 코드 변경의 근거가 되는 결정·액션이 확정된 경우에만 간결 요약으로 게시합니다.

## 8) 부록: 최소 요약 템플릿 (Issue/PR 설명)

```text
Summary
- (3~5줄 요약)

Decision
- (핵심 1문장)
- 근거 1~2: (내부 링크)

Evidence
- 내부 리포트/대시보드/로그 링크

Action Items
- [ ] 항목 A — @owner — YYYY-MM-DD
- [ ] 항목 B — @owner — YYYY-MM-DD
```
