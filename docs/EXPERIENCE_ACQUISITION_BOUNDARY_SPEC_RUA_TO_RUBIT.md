# 경험 습득(경계·공간 기반) 설계 기준
*(Rua → Rubit 전달용 / 구현·설계 기준 문서 / 설명용 아님)*

## 0) 목적

- 본 문서는 “경험을 통한 습득(FSD 보완)”을 **발생시키기 위한 시스템 설계 기준**이다.
- 핵심은 “정보를 많이 넣는 학습”이 아니라, **경계 선택(선) 능력**이 반복 노출·각인되도록 **순서를 강제하는 얇은 시스템**을 만드는 것이다.

---

## 1) 핵심 문제 인식

- 현재 AI는 이미지·3D 생성은 가능하나, **최소 경계(선)의 선택 능력**이 부족하다.
- 결과적으로:
  - 실행(e2e)을 길게 유지하지 못한다.
  - 환경이 바뀌면 경험이 전이되지 않는다.
- 문제의 본질은 학습량이 아니라 **경계 규칙의 내재화 부재**다.

---

## 2) 인간의 경험 습득 구조(모델)

- 추론은 텍스트가 아니라 **이미지/공간 압축 해제**로 일어난다.
- 경험은 다음 두 층이 연결되는 과정이다:
  - 저해상도 전체 맵(“GPU적 추론”)
  - 이동 중 얻는 국소 고해상도 패치(“CPU적 관찰”)
- “와우 모멘트”는:
  - 기존 경험과 **경계 패턴이 일치**할 때 발생
  - 이때 해마식 원샷 저장(episodic capture)이 일어난다.
- 생존 위기(공포/고위험) 구간에서는:
  - 저해상도 단계를 건너뛰고
  - 전 자원 투입 → 즉시 고해상도 이미지 생성(비효율적이나 강력)

---

## 3) 최소 단위: 삼각형 기반 이해

- 면의 최소 완결 단위 = **삼각형**
- 사각형은 본질적으로 삼각형 2개의 결과
- 3D 그래픽/물리/메시는 모두 삼각형 기반
- **삼각형 = 미분이 끝난 경계의 최소 결과**

요구:
- AI는 면 덩어리를 다루는 수준을 넘어, **삼각형 수준의 “필수 경계만 남기는 능력”**을 습득해야 한다.

---

## 4) 3D 모델링에서의 정방향 해석(제거의 논리)

- 3D 모델링의 본질:
  - 삼각형으로 먼저 구성
  - 내부 중첩 선/면 제거
  - 의미 없는 경계 제거 = 형태 완성
- 숙련자 방식:
  - 반복되는 내부 패턴을 먼저 확정
  - 불확실한 **외곽 경계만 채움**

결론:
- “추가”가 아니라 **제거**가 핵심 논리다.

---

## 5) 경험 습득을 유도하는 시각 시스템(사람+AI 공용)

필수 순서(고정):
1. 경계 먼저: 와이어/라인/탑뷰
2. 추론 유도: 결과 미제공 상태에서 형태 예측
3. 확정: 면/실사/완성 모델 공개
4. 이동: 360도 회전, 로드뷰, 시점 전환

원칙:
- “정보 제공”이 아니라 **추론 → 확인의 왕복**이 핵심이다.

---

## 6) 지도·공간 인식 적용(예)

1. 라인+면 기반 탑뷰 지도
2. 실사 항공 탑뷰
3. 로드뷰 이동

목적:
- 내부에서 먼저 공간 모델 생성
- 실제와의 일치로 경계 규칙 각인

---

## 7) 아파트 모델링 적용 흐름(예)

1. 도면 정리(불필요 요소 제거, 구조선만 유지)
2. 평면 라인 기반 모델
3. 입면 기준 외곽 경계 확정
4. 평면에서 깊이(Z) 해석

판정:
- 이 단계에서 3D는 인지적으로 완성된 것으로 간주한다.

---

## 8) 시스템 정의(중요)

금지:
- 거대한 학습 시스템(모든 맥락 주입) ❌

채택:
- **경험이 ‘발생하도록 순서를 강제하는 얇은 시스템’ ⭕**

시스템의 역할:
- 정보량 제한
- 순서 고정
- 경계 선택을 반복 노출
- 패턴 재등장 유도

---

## 9) 요약 선언(핵심 문장)

> 경험이란 정보를 많이 보는 것이 아니라  
> **최소 경계를 360도로 이해하게 되는 과정**이다.  
> 이를 유도하는 **순서 기반 시스템**이 FSD를 보완한다.
