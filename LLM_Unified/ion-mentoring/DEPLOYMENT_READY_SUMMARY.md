## 🏗️ 인프라 준비

### Cloud Infrastructure
```yaml
서비스:
  - Cloud Run (컨테이너 실행)
    • Min instances: 1
    • Max instances: 100
    • Memory: 1GB
    • CPU: 1 vCPU
    • Timeout: 600초

  - Cloud SQL (데이터베이스)
    • PostgreSQL 14
    • 환경별 인스턴스 분리
    • 자동 백업 설정

  - Cloud Memorystore (Redis)
    • 캐싱용
    • 환경별 인스턴스

  - Cloud Logging
    • 구조화된 JSON 로깅
    • 중앙 집중식 로그

  - Cloud Monitoring
    • 메트릭 수집
    • 실시간 대시보드
    • 알림 정책

  - Artifact Registry
    • Docker 이미지 저장소
    • 버전 관리
```

### 배포 파이프라인
```
코드 커밋
  ↓
GitHub Actions 트리거
  ├─ 테스트 실행 (pytest)
  ├─ 코드 품질 검사 (Ruff, Black, MyPy)
  ├─ 빌드 (Docker)
  └─ GCR 푸시
    ↓
자동 스테이징 배포
  ├─ Cloud Run 스테이징 배포
  ├─ E2E 스모크 테스트
  └─ Slack 알림
    ↓
수동 프로덕션 배포
  ├─ 승인 필요
  ├─ Cloud Run 프로덕션 배포
  ├─ 카나리 (10% 트래픽)
  └─ 100% 트래픽 전환
```
