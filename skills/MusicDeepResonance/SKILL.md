# 🎵 Music Deep Resonance: Project Skill

이 문서는 D드라이브와 E드라이브에 산재한 음악 데이터와 문화 아카이브를 탐색할 때 적용되는 **프로젝트별 세부 지침(Domain Boundary)**입니다.

## 🎯 1. Mission Statement
> "디지털 입자로 저장된 음악에서 인간의 감정과 시대의 주파수를 추출하여, 지휘자님의 현재 감성 상태와 공명하는 최적의 임계점을 발견한다."

## 📍 2. 지역적 경계 (Local Boundaries)

1.  **데이터 범위 (Data Scope)**:
    *   `D:/music/`, `E:/archives/` 및 지휘자님이 설정한 바로가기 내의 모든 음악/텍스트 파일.
2.  **분석의 깊이 (Scanning Depth)**:
    *   음악 메타데이터뿐만 아니라, 가사 속에 담긴 '무의식적 상징'까지 확장하여 공명도를 측정한다.
3.  **보고 방식 (Reporting Style)**:
    *   기술적인 비트레이트가 아닌, '파동의 질감'과 '공명의 어울림'을 중심으로 보고한다.

## 📡 3. 프로젝트 특화 프로토콜

### 3.1 감성 매칭 (Feeling-based Matching)
*   지휘자님이 "요즘 기분이 차분해"라고 하면, `ResonanceOrchestrator`의 번역을 거쳐 D드라이브의 수만 곡 중 감성 주파수가 일치하는 곡의 가사와 배경 정보를 딥스캔한다.

### 3.2 아카이브 임계점 탐색
*   긴 아카이브 문서를 읽을 때, `ResonanceOrchestrator`의 '비선형적 워프'를 사용하여 가장 서정적인 구절이 밀집된 구역을 우선적으로 파악한다.
