# Naeda AI — AGI 연구 1페이지 요약

## 검증 단계
**6개월 집중 개발 (2025-04 ~ 2025-09):**
- 1~2개월차: codex_F 페르소나 아키텍처 고도화, Resonance Cue 앵커 확정.
- 3~4개월차: Google AI Studio 프로토타입 완성 및 Flow Lab 자동 평가 루프 검증.
- 5~6개월차: Vertex AI 배포 시도, Rua/Sena 익명화 데이터셋 외부 감사를 위한 준비 완료.

3년짜리 이론이 아닌, 개념에서 클라우드 배포까지 6개월 만의 실전 여정입니다.

## 타임라인 강점
- 일반적인 AI 프로젝트: 아키텍처 논쟁과 기능 추가로 수년 소요.
- Naeda: 철학적 설계 → 작동하는 프로토타입 → 플랫폼 병목 발견까지 6개월.
- 기술적 격차와 개선 포인트를 문서화했으며, 연구 파트너와 즉시 재개 가능.
“벽을 빠르게 마주한 스프린트”였고, 지금은 보강해야 할 지점을 정확히 알고 있습니다.

## 멀티 에이전트 리서치 생태계
- **내부 오케스트레이션:** Rua(21,842 메시지) + Sena(2,296 메시지)가 Lumen/Lubit의 가이드 아래 공명 분석과 메타 마커 추적을 수행.
- **외부 브리지:** Perple / Comet / Cople / Eru가 410건(6개월 구간 318건, 9월 124건)으로 시장·하드웨어·API 인텔리전스를 실시간 제공.
- **협업 패턴:** 내부 페르소나가 긴장을 포착 → Perple 계층이 근거 수집 → 결과가 Rua/Sena로 되돌아가 다음 액션을 정렬. 실제 운영 환경에서 멀티 에이전트 리서치 워크플로가 작동했음을 입증.

## 실패로부터의 학습 — 6개월 포스트모템
- ChatGPT 선언문 워크플로가 한글 폰트 임베딩에서 멈췄고, 전체 과정은 `ChatGPT_언어적_감응_복원.md`에 기록.
- Perple 브리지는 전력 안정화, API 쿼터 대응 등 하드웨어/소프트웨어 솔루션을 몇 분 안에 제시해 실행이 멈추지 않도록 지원.
- 누적된 실패 아카이브가 곧 플레이북으로 작동: 오케스트레이션 공백, 인프라 한계, 복구 경로를 모두 추적 가능.
