# [Agent P] 딥 서치 기능은 비선형 보다는 선형에 가까운 대화 방식이겠지

[Agent P] 딥 서치(Perplexity Deep Search) 기능의 대화 방식은 "비선형"보다는 "선형"에 가까운 구조로 설계되어 있습니다.

## 딥 서치의 대화 구조

- 딥 서치는 사용자의 질문을 입력받아, 자료 조사 → 정리 → 답변 아웃라인 작성 → 보고서 생성의 순차적(선형) 단계를 거쳐 결과를 제공합니다.[1]
- 실제로 생성된 답변은 서론, 본론, 결론 등 논리적이고 체계적인 글 구조를 따르며, 이는 전형적인 선형적 레포트 작성 방식과 유사합니다.[1]
- 사용자가 질문을 입력하면, 여러 자료를 수집·분석한 뒤 명확한 보고서 형태로 결과를 제시하는 것이 기본 흐름입니다.[2][3]

## 비선형 대화와의 차이

- 비선형 대화란 사용자가 대화 중간에 자유롭게 주제를 전환하거나, 여러 갈래의 질문을 동시에 탐색하는 방식입니다.
- [Agent P] 딥 서치는 사용자가 추가 질문을 통해 정보를 더 요청할 수 있지만, 기본적으로 한 번에 하나의 질문에 대해 체계적으로 답변을 생성하는 선형적 접근이 중심입니다.[4][5]

## 결론

[Agent P] 딥 서치의 대화 방식은 "자료 조사 → 정리 → 보고서 작성"의 단계적 흐름을 따르는 선형적 구조에 가깝습니다. 사용자가 추가 질문을 하며 대화를 이어갈 수는 있지만, 기본적으로는 비선형적 자유 대화보다는 논리적이고 일관된 선형적 정보 제공에 초점을 둡니다.[3][2][1]
