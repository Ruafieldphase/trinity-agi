- **Resonance (공명)**: 두 주파수가 일치할 때 진폭 극대화

**우리 시스템에서**:

```

  - 사용자 요청
  - 에러 발생
  - 시간대 변화
  - 부하 증가

공명 조건 (Resonance Condition):
  외부 자극의 주파수 ≈ 내재 리듬
  → 자연스러운 전환 (특이점)
  → 최소 에너지로 상태 변화
  
역공명 (Anti-Resonance):
  외부 자극의 주파수 ≠ 내재 리듬
```

## 🎭 맥락 기반 (Context-Driven) 리듬

### 고정된 순서가 아니라 맥락에 따라

**Adaptive Rhythm Orchestrator**가 4가지 모드를 동적으로 전환:

#### 🟢 NORMAL Mode (접힘-펼침 균형)

- CPU < 80%, 메모리 < 85%
- 정상적인 접힘-펼침 리듬
- 모든 시스템 활성화

#### 🟡 BUSY Mode (펼침 우세)

- CPU 70-90%
- 빠른 폴링 (50ms)
- 외부 작업 중심 (학습 일시 중지)
- **펼침 단계 유지**

#### 🔴 EMERGENCY Mode (강제 접힘)

- CPU > 90% 또는 에러율 > 10%
- Direct Mode (큐 우회)
- 최소 레이어만 활성화
- **강제 접힘으로 전환** (생존)

#### 🔵 LEARNING Mode (깊은 접힘)

- 밤 시간 (22:00-06:00) + CPU < 50%
- BQI 학습 최대화 (100%)
- 내부 통합 중심
- **깊은 접힘 단계** (학습)
