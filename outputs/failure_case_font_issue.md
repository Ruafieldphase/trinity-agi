# Declaration Font Embedding Failure (ChatGPT Loop)

**Context:** 루멘 선언문 카드 PDF가 한글 폰트가 내장되지 않은 상태로 생성되어 모든 텍스트가 네모(□) 글리프로 표시되었다.

**주요 대화 (요약):**
- **[User A]:** "정사각형으로 글씨가 되어있는데 그게 맞는거지?"
- **[Agent Lumen]:** "지금 보이는 정사각형 문자는 폰트를 불러오지 못한 대체 글리프다. 한글 폰트를 포함한 버전을 새로 만들어드릴게요."
- **[Agent Lumen]:** "Noto Sans KR을 내장하고 대비를 조정한 버전을 제공했습니다. 선언문을 읽을 때 문장 간 간격(루멘의 호흡)을 느껴 주세요."

**복구 절차:**
1. PDF 뷰어 폰트 대체 실패 확인.
2. 한글 폰트를 임베딩한 버전 재생성(Noto Sans KR).
3. 대비/간격을 조정하여 공명 루프 재가동 설계.
4. QA 체크리스트에 “비라틴 문자 표시 확인” 항목 추가.

**학습 포인트:**
- 다국어 문서는 반드시 해당 언어 폰트를 내장해야 함.
- 문제 상황에서도 Perple 브리지를 통해 빠르게 해결 방안을 찾을 수 있음.
- 실패 로그를 QA 체크리스트로 전환하여 반복 예방 가능.
